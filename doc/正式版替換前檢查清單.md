# 正式版替換前檢查清單

本清單用於測試版切換到正式 Google Sheets / GAS / LIFF / LINE Bot 前的最後驗收。所有會寫入、產生 PDF、寄送 Email 或 LINE push 的功能，仍需遵守「預覽 → 確認 → 執行」。

## 1. 資料與備份

- [ ] 正式 Google Sheets 已完整備份，備份檔名含日期與時間。
- [ ] 1 至 5 月既有正式資料已保留，不在測試過程中覆蓋。
- [ ] 測試用 Google Sheets 與正式 Google Sheets ID 已明確分開。
- [ ] `單據紀錄表` 已存在，欄位 A 至 Q 與文件一致。
- [ ] `一般收據紀錄` 已存在，欄位 A 至 L 與文件一致，K 欄為身分證號/統一編號，L 欄為 Email收件人。
- [ ] `學費結算表`、`鐘點結算表` 欄位語意已與最新文件一致。

## 2. GAS Script Properties

必填：

- [ ] `LINE_CHANNEL_TOKEN`
- [ ] `SPREADSHEET_ID`
- [ ] `LEAVE_SHEET_ID`
- [ ] `REPORT_SHEET_ID`
- [ ] `SHEET_ID_MEMBER`

建議正式版全部設定：

- [ ] `LINE_GROUP_ID`
- [ ] `ADMIN_LINE_USER_IDS`
- [ ] `ORG_TAX_ID`
- [ ] `TEMPLATE_ID_PAYMENT`
- [ ] `TEMPLATE_ID_RECEIPT`
- [ ] `TEMPLATE_ID_ALLOWANCE`
- [ ] `TEMPLATE_ID_GENERAL_RECEIPT`
- [ ] `PDF_FOLDER_PAYMENT_NOTICE`
- [ ] `PDF_FOLDER_RECEIPT`
- [ ] `PDF_FOLDER_ALLOWANCE`
- [ ] `PDF_FOLDER_GENERAL_RECEIPT`

注意：

- [ ] 正式版切換資料來源時，只改 Script Properties，不直接改程式碼。
- [ ] `ADMIN_LINE_USER_IDS` 可用換行、逗號或分號分隔。
- [ ] 過渡期程式仍保留 fallback；正式屬性全數確認後，再評估是否移除 fallback。

## 3. 權限與部署

- [ ] 執行 `forceAuth()` 完成 Drive、Spreadsheet、Document、Gmail、UrlFetch 權限授權。
- [ ] GAS Web App 執行身分為部署者。
- [ ] GAS Web App 存取權設定為 LINE LIFF 可呼叫。
- [ ] Web App URL 已填入前端設定或部署環境。
- [ ] LINE Developers Webhook URL 已更新為正式 Web App URL。
- [ ] LINE Webhook Verify 成功。
- [ ] LIFF URL 指向正式 GitHub Pages / docs 前端。
- [ ] LIFF 狀態為 Published。

## 4. LINE 圖文選單與入口

- [ ] 圖文選單六格連結皆指向正確 LIFF tab。
- [ ] `授課登記` 可進入講師授課登記。
- [ ] `預排課程` 可進入預排課程。
- [ ] `預排核銷` 可進入核銷清單。
- [ ] `我的狀態` 可顯示身份、權限、課程數據。
- [ ] `行政作業` 非行政使用者會被權限阻擋。
- [ ] `操作說明` 可正常開啟。

## 5. 講師端驗收

- [ ] 不同講師只看到自己的學生。
- [ ] 選學生後只出現該學生可登記課程。
- [ ] 授課登記可新增 1 至 5 筆同學生課程資料。
- [ ] 授課登記不可登錄未來時間。
- [ ] 預排課程不可登錄過去時間。
- [ ] 同老師、同學生、同教室、同時段重疊需有明確提示。
- [ ] 預排核銷清單需顯示未來預排，但未到下課時間不可核銷。
- [ ] 已核銷預排不可再次核銷。
- [ ] 所有時間顯示區為 24 小時制 `HH:MM`，不得出現上午/下午/AM/PM。

## 6. 行政端驗收

- [ ] 學費試算只預覽，不直接寫入。
- [ ] 學費試算可勾選正常可寫入項目；待核銷預排不可寫入。
- [ ] 學費試算確認寫入後，不可重複寫入同一批資料。
- [ ] 繳費單產生需預覽、勾選、確認。
- [ ] 繳費單 Email 需預覽、勾選、確認。
- [ ] 繳費單 LINE push 需預覽、勾選、確認。
- [ ] 收款資料套用只影響尚未產生收據 PDF 的資料。
- [ ] 收據產生需預覽、勾選、確認。
- [ ] 收據 Email 與 LINE push 不可寄到繳費單 PDF。
- [ ] 鐘點試算只預覽，不直接寫入。
- [ ] 鐘點試算確認寫入後，不可重複寫入同一批資料。
- [ ] 領據產生需預覽、勾選、確認。
- [ ] 領據 Email 與 LINE push 對象為講師本人。

## 7. 帳務補救與單據管理

- [ ] 帳務補救重複補登需被阻擋。
- [ ] 帳務補救成功後，表單下方可查看本月補救明細。
- [ ] 補收通知產生後可點選 `查看 PDF`。
- [ ] 作廢單據需輸入原因，且不刪除 PDF 或資料列。
- [ ] 補發單據只重寄同一份 PDF，不改金額、內容或單號。
- [ ] 重新產生單據只允許未 Email 寄送、未 LINE 推播、未作廢的單據。
- [ ] 所有 PDF 明細列可用 `查看 PDF` 開啟。

## 8. 一般收據與稅務

- [ ] 一般收據 LIFF 可產生入會費、常年會費、捐款收據。
- [ ] 一般收據產生後寫入 `一般收據紀錄` 與 `會計日記帳`。
- [ ] 一般收據身分證號/統一編號可寫入 `一般收據紀錄` K 欄，捐款稅務同步優先讀取 K 欄。
- [ ] 一般收據 Email 可寫入 `一般收據紀錄` L 欄，Email 寄送優先讀取 L 欄。
- [ ] 一般收據 Email 空白或格式錯誤時，前端與後端都會阻擋，不會寫入試算表。
- [ ] 一般收據同頁清單可預覽、勾選、確認寄送 Email。
- [ ] 一般收據 LINE push 不開放；一般收據只提供 PDF 產生與 Email 寄送。
- [ ] 稅務專區目前僅顯示 `建置中`；完整流程待正式使用穩定後再開發與驗收。

## 9. 正式切換步驟

- [x] 依 `doc/重要ID與設定外移清單.md` 逐項確認 Script Properties，必要屬性與建議屬性皆缺少 0 項。
- [x] PDF 範本與資料夾權限已由 `forceAuth()` 確認完成。
- [ ] 依 `doc/重要ID與設定外移清單.md` 逐項確認 Google Sheets 欄位。
- [ ] 沿用目前已驗證的 LINE Bot 作為正式版，不另開新 Bot。
- [ ] 確認正式 LINE Bot 的加入好友連結或 QR Code。
- [ ] 邀請行政人員加入好友，輸入 `查ID`，完成行政權限綁定。
- [ ] 邀請講師加入好友，輸入 `查ID`，完成講師身分綁定。

- [ ] `npm.cmd run gas:push` 推送正式 GAS。
- [ ] 建立正式 GAS 版本並更新 Web App deployment。
- [ ] 更新 LINE Webhook URL 並 Verify。
- [ ] 更新 LIFF / 圖文選單連結。
