# 測試清單與驗收規格

最後更新：2026-06-05

## 通用驗收原則

- 所有會寫入、產生單據、寄送 Email、LINE push 的功能，都必須先顯示預覽。
- 未確認前不可寫入。
- 權限不足者不可看到或執行行政功能。
- 前端擋住之外，後端也必須檢查權限。
- 錯誤情境必須有明確提示，不可只有無反應。

## LIFF 講師端

### 我的狀態

- 顯示身份別。
- 顯示權限。
- 顯示本月已登記授課。
- 顯示待核銷預排。
- 顯示可登記學生。
- 顯示可登記課程。
- 點選數字後顯示對應資料。

### 授課登記

正常流程：

- 選學生後，只出現該學生課程。
- 日期、時間、課程完整後可送出。
- 成功後寫入授課紀錄。

錯誤流程：

- 空欄位不可送出，需顯示提示。
- 未來時間不可登記為已授課。
- 結束時間早於開始時間不可送出。
- 課程時數過長不可送出。
- 同批新增時間重疊不可送出。
- 同老師同時段重疊不可寫入。
- 同學生同時段重疊不可寫入。

### 預排課程

正常流程：

- 可登記未來課程。
- 成功後寫入預排紀錄。
- 成功後出現在預排核銷清單。

錯誤流程：

- 過去時間不可預排。
- 完全重複預排不可重複寫入。
- 同老師、同學生、同時段重疊需提示。

### 預排核銷

正常流程：

- 未來預排可顯示在清單。
- 課程時間結束後可核銷。
- 有上課後轉入或更新授課紀錄。
- 未上課後取消預排並進入帳務規則。

錯誤流程：

- 未來時間不可核銷。
- 同一筆預排核銷後，再點一次應被擋下。
- 無權限或非本人資料不可核銷。

## 行政端

### 試算

- 學費試算先顯示預覽。
- 鐘點試算先顯示預覽。
- 未確認不可寫入。
- 確認後才寫入試算結果。
- 重複試算不可產生重複錯誤資料。
- 在正式預覽 API 接上前，試算入口不可直接呼叫寫入型後端任務。
- 前端預覽卡需顯示目前狀態與停用的下一步按鈕。
- 切換行政 Tab 後，上一個預覽卡應清除。
- `adminPreview` API 必須是只讀 API，不可寫入任何試算或單據資料。
- `adminPreview` API 權限不足時需回傳錯誤，不可提供行政預覽。
- `adminPreview` API 可回傳資料來源統計，但不得產生正式結算資料。
- 前端預覽卡需顯示資料來源統計區塊。

### 繳費單

- 產生前顯示預覽。
- 預覽需包含學生、課程、月份、預收/後收、補收/退費、總金額。
- 確認後才產生繳費單。
- 寄送 Email 前需顯示收件人與內容摘要。
- LINE push 前需確認家長 LINE 綁定。
- 已產生單據可查看。
- 後續測試重新產生、作廢、補發。
- 在正式預覽 API 接上前，繳費單入口只可顯示預覽或暫停提示，不可直接產生或寄送。

### 收據

- 產生前顯示預覽。
- 確認後才產生收據。
- 寄送 Email 前需確認收件人。
- LINE push 前需確認家長 LINE 綁定。
- 一般收據需測入會費、常年會費、捐款。
- 在正式預覽 API 接上前，收據與一般收據入口只可顯示預覽或暫停提示，不可直接產生或寄送。

### 領據

- 產生前顯示預覽。
- 預覽需包含講師、月份、授課紀錄、鐘點費、補充項目。
- 確認後才產生領據。
- LINE push 前需確認講師 LINE 綁定。
- 在正式預覽 API 接上前，領據入口只可顯示預覽或暫停提示，不可直接產生或寄送。

### 帳務補救

- 可選補收或退費。
- 可選原錯誤月份與處理月份。
- 選學生後，只出現該學生課程。
- 選課程後，自動帶出單價。
- 選時間後，自動計算時數。
- 可選關聯原單號。
- 建立前需顯示補救預覽。
- 確認後才寫入補救紀錄。
- 不可直接改舊繳費單。

## 時間欄位

- 手機顯示區為 `HH:MM`。
- 電腦顯示區為 `HH:MM`。
- 電腦自訂面板小時為 `06-18`。
- 分鐘為 `00 / 15 / 30 / 45`。
- 不可顯示 `上午`、`下午`、`AM`、`PM` 作為表單顯示值。

## 回歸測試

每次修改 LIFF 後至少測：

- 講師身份登入。
- 行政身份登入。
- 非行政無法進入行政作業。
- 授課登記一筆成功。
- 預排課程一筆成功。
- 未來預排不可核銷。
- 行政高風險功能不可直接寫入。
