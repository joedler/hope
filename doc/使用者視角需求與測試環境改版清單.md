# 使用者視角需求與測試環境改版清單

更新日期：2026-06-02

## 1. 目前共識

空空BOT 已自 2026 年 1 月開始正式使用，1 至 5 月的授課、請假、財務與單據相關資料已累積在正式 Google Sheets 中。因此本次改版不應推翻重做正式系統，而應採用：

- 保留正式 Google Sheets 既有資料與欄位邏輯。
- 保留既有 LINE 文字指令作為過渡期 fallback。
- 在現有 repo 基礎上繼續重構。
- 另開測試環境平行開發與驗證。
- 等低峰期確認穩定後，再切換正式 LINE Webhook / LIFF / Rich Menu。

簡單結論：

> 不是重新開一個完全新專案，而是在現有專案基礎上升級；但測試環境必須獨立。

## 2. 目前使用者角色與功能

### 講師

目前功能：

- 授課登錄。
- 預排課程。
- 預排核銷。
- 刪除最後一筆授課或預排紀錄。
- 請假。
- 查詢選單、說明或自身操作資訊。

目前操作方式：

- 主要透過 LINE 文字指令，例如 `登錄`、`預排`、`核銷`、`請假`。
- 系統回覆 Flex Message 選單或卡片。
- 使用者再輸入日期、時間、學生或課程資訊。
- GAS 檢查重疊、計算時數或寫入 Google Sheets。

主要痛點：

- 每月可能只操作一次，容易忘記指令。
- 日期與時間格式容易輸入錯。
- 對話流程中斷後，使用者不知道如何回到正確步驟。
- 使用者難以判斷錯誤來自輸入、資料表還是程式計算。

### 助理 / 行政

目前功能：

- 代替講師登錄、預排、核銷。
- 學費試算。
- 鐘點費試算。
- 產生繳費單。
- 查詢繳費單。
- 開收據、寄收據。
- 開領據、寄領據。
- 發公告。
- 查 ID、查群 ID。

目前操作方式：

- 透過 LINE 文字指令搭配 Flex Message。
- 部分流程使用快取狀態，要求使用者接著輸入月份、姓名或確認操作。

主要痛點：

- 指令多且相似，助理不易記憶。
- 月份、姓名、格式輸入錯會造成流程失敗。
- 財務與單據功能風險較高，不適合直接一鍵寫入或寄送。
- 批次功能若缺少預覽與異常清單，容易在錯誤資料上繼續執行。

### 財務 / 管理者

目前功能：

- 會計記帳。
- 財務三表。
- 免稅試算。
- 年度扣繳。
- 捐款申報。
- 一般收據。
- 批次產生或寄送單據。

主要痛點：

- 計算結果仰賴 Google Sheets 欄位、公式與 GAS 程式一致。
- 高風險操作需要清楚的預覽、確認與操作紀錄。
- 不應讓未授權者看到或執行敏感財務功能。

## 3. 未來建議操作方式

最佳方向為：

> LINE Rich Menu 作為入口，LIFF Web App 作為主要操作介面，LINE 訊息作為通知與確認結果。

### 講師端建議

講師不再需要記文字指令，改由 Rich Menu 進入 LIFF：

- 授課登記。
- 預排課程。
- 預排核銷。
- 請假申請。
- 我的紀錄。
- 說明。

LIFF 表單應提供：

- 學生下拉選單。
- 課程下拉選單。
- 日期選擇器。
- 開始與結束時間選擇器。
- 正式授課 / 預排切換。
- 送出前摘要。
- 後端重疊檢查。
- 成功或失敗訊息。

### 助理 / 行政端建議

行政使用 LIFF 工作台，不再依賴大量文字指令。

建議分區：

- 課程代操作。
- 月結試算。
- 單據作業。
- 講師與學生查詢。
- 公告工具。
- 系統工具。

行政操作流程建議：

1. 選擇月份、講師、學生或任務。
2. 系統顯示試算或操作預覽。
3. 顯示異常資料，例如缺 email、缺金額、缺講師資料。
4. 助理二次確認。
5. 系統才寫入 Sheet、產生 PDF 或寄送 Email。
6. LINE 或 LIFF 顯示執行結果與錯誤明細。

### 財務與高風險功能建議

高風險功能不可第一版就直接一鍵實寫。

建議採用：

- 先試算。
- 再預覽。
- 顯示異常清單。
- 二次確認。
- 寫入或寄送。
- 留操作紀錄。

適用功能：

- 學費試算。
- 鐘點費試算。
- 產生繳費單。
- 寄收據。
- 寄領據。
- 年度扣繳。
- 捐款申報。
- 免稅試算。

### 測試 GAS 專案應如何建立

建議建立「獨立 Apps Script 專案」，不要建在複製 Sheet 的「擴充功能 -> Apps Script」底下。

原因：

- 獨立 GAS 更適合使用 `clasp` 管理與部署。
- 測試 GAS、正式 GAS、測試 Sheet、正式 Sheet 的邊界更清楚。
- 未來切換 Webhook 或部署版本比較乾淨。
- 測試專案可重建、重推、重部署，不會綁死在單一試算表容器內。
- 若之後產品化或複製給其他客戶，獨立 GAS 架構更容易複製。

複製 Sheet 仍然作為測試資料庫，只要把測試 Sheet ID 設定到測試 GAS 的 Script Properties 即可。

## 6. 需要使用者授權我才能做的項目

- 執行 `npm install` 下載依賴。
- 執行 `npm run gas:build` 或 `npm run gas:check` 做建置驗證。
- 執行 `npm run gas:push` 推送 GAS。
- 使用 `clasp` 建立、部署或更新 Apps Script。
- 設定 Git safe.directory。
- commit / push 到 GitHub。
- 呼叫真實 GAS Web App URL 做 API 測試。

- 設定測試 GAS 的 Script Properties。
- 建立測試 LINE Bot Channel。
- 建立測試 LIFF App 並取得 LIFF ID。
- 設定測試 Webhook URL。
- 啟用或確認 GitHub Pages。

- 請假功能是講師自助，還是助理代操作為主。
- 講師綁定是否只用姓名，或需加手機末三碼 / 管理員審核。
- `MAGIC_KEYWORDS` 是否改由 Google Sheets 設定。
- 行政專區第一版是否只做預覽，不直接寫入。
- 財務與單據批次功能的確認流程。
- 1 至 5 月既有資料是否需要建立欄位檢查與資料一致性報告。
