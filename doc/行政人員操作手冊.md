# 行政人員操作手冊

最後更新：2026-06-13

## 操作原則

行政端所有高風險動作都必須先看預覽，再確認執行。

高風險動作包含：

- 寫入試算結果
- 產生繳費單
- 產生收據
- 產生領據
- 寄送 Email
- LINE push
- 建立帳務補救紀錄
- 作廢、補發、重新產生單據

## 行政端 Tab

### 試算

用途：計算指定月份的學費與講師鐘點。

流程：

1. 選擇月份。
2. 點選學費試算或鐘點試算。
3. 查看預覽結果。
4. 確認無誤後才寫入。

現況：

- 前端會優先讀取只讀型 `adminPreview` API。
- `adminPreview` 不會寫入、產生單據、寄送 Email 或 LINE push。
- 若出現「找不到分頁」或「讀取失敗」，不要繼續進行結算或產生單據，需先排查 Google Sheets 分頁名稱、權限或欄位設定。
- 學費試算預覽會先彙總待試算授課、預收下月預排、預收核對差異與帳務補救金額。
- 預收課程若尚有未核銷預排，預覽會列為待核銷並暫不計入退費；需先完成預排核銷，才可正式寫入學費結算。
- 若預覽提示該月份已有學費結算資料，代表這可能是舊月份回看情境；回看舊繳費單應使用「已產生單據」，不要重新試算。
- 鐘點試算預覽會先彙總待試算授課、講師時數、應付總額、扣繳稅額、補充保費與實發金額。
- 若該月份已有鐘點結算資料，系統會顯示既有結算摘要並停用確認寫入；不可為同月份同一批鐘點重複試算寫入。
- 已寫入的學費或鐘點再次預覽時，請視為回看狀態，不可再確認寫入。學費試算會帶入繳費單狀態、PDF 連結與待核銷預排；若已有 PDF，可直接點 `查看 PDF`。
- 若預覽提示該月份已有鐘點結算資料，代表這可能是舊月份回看情境；回看舊領據或結算應使用「已產生單據 / 領據紀錄」，不要重新試算。
- 學費試算與鐘點試算可於預覽確認後按「確認寫入」；系統會重新試算並寫入結算表，同時執行重複寫入防呆。
- 學費試算寫入後，請接續檢查繳費單預覽；鐘點試算寫入後，請接續檢查領據預覽。
- 學費試算預覽會以課程列顯示狀態；待核銷預排只供核對，在寫入模式下不可勾選。
- 繳費單可於預覽確認後按「確認產生繳費單」，系統會產生 PDF 並回寫學費結算表 PDF URL 與狀態。
- 學費結算寫入後，系統會顯示繳費單狀態；先點選要執行的步驟，畫面只允許該步驟符合條件的資料勾選。
- 鐘點結算寫入後，系統會在同一個試算預覽區顯示領據預覽；領據產生仍待後續開放。
- 切換行政 Tab 時會清除上一個預覽，避免誤讀不同功能的內容。

### 繳費單

用途：產生並寄送學生繳費通知。

流程：

1. 選擇月份與學生範圍。
2. 產生繳費單預覽。
3. 檢查學生、課程、預收/後收、補收/退費、總金額。
4. 確認後產生繳費單。
5. 可寄送 Email 給家長。
6. 後續新增 LINE push 給家長個人 LINE。
7. 可查看已產生單據。
8. 後續增加重新產生、作廢、補發。

現況：

- 繳費單預覽會讀取學費結算表，顯示學生、單據編號、總金額、課程數與 PDF 產生狀態。
- 繳費單預覽不會重新試算學費，也不會產生 PDF、寫入 PDF URL、寄送 Email 或 LINE push。
- 確認產生繳費單後，只會產生 PDF 與回寫狀態，不會寄送 Email 或 LINE push。
- 可勾選單一或多位學生產生繳費單；未勾選者不會產生。
- 若該月份繳費單已全部產生，確認按鈕會停用，避免重複產出。
- 若看不到繳費單資料，需先確認該月份是否已完成學費試算確認寫入。

### 帳號與權限

- 「解除 LINE 帳號綁定」只放在「我的狀態」底部。
- 解除綁定屬低頻高風險操作，授課登記、預排、核銷與行政作業頁不重複顯示。

### 收據

用途：針對已繳費資料產生與寄送收據。

流程：

1. 選擇月份或繳費單。
2. 產生收據只讀預覽。
3. 確認付款資訊、學生、金額、收款方式、收據類別、收款日期、PDF 與寄送狀態。
4. 確認後產生收據。
5. 可寄送 Email 給家長。
6. 後續新增 LINE push 給家長個人 LINE。

一般收據：

- 適用入會費、常年會費、捐款。
- 需要預覽與寄送功能。
- 後續可再依稅務需求擴充。

### 領據

用途：針對講師鐘點費產生與寄送領據。

流程：

1. 選擇月份與講師。
2. 產生領據只讀預覽。
3. 檢查授課紀錄、預排核銷、補充鐘點、總金額、領據編號、PDF 與寄送狀態。
4. 確認後產生領據。
5. 可寄送 Email 給講師。
6. 後續新增 LINE push 給講師個人 LINE。

### 帳務工具與帳務補救

用途：處理漏登、短收、超收、補收、退費。

原則：

- 不直接修改舊繳費單。
- 必須建立補救紀錄。
- 必須關聯原繳費單。
- 必須在後續月份反映補收或退費。
- 補收/退費要併入處理月份月結，不重新產生或覆蓋原繳費單；必要時可另產生補救明細/補收通知作為說明。

流程：

1. 選擇補收或退費。
2. 選擇原錯誤月份。
3. 選擇處理月份。
4. 選擇學生。
5. 選擇課程。
6. 填入原上課日期與時間。
7. 系統帶出時數、單價、預覽金額。
8. 選擇關聯原單號。
9. 填寫原因。
10. 預覽補救影響。
11. 確認後建立補救紀錄。

補救明細/補收通知流程：

1. 在帳務工具分頁選擇處理月份。
2. 點選「預覽補救明細」。
3. 檢查待處理補收的學生、金額、筆數與關聯原單號。
4. 確認後點選「確認產生」。
5. 系統產生補救明細/補收通知 PDF，單號格式為 `ADJ_YYYY_MM_###`。
6. 系統回寫學費調整紀錄表的補收單號、補收 PDF、補收單狀態，並將狀態改為已產生補收單。

注意：

- 補救明細/補收通知只處理調整類型為補收、狀態為待處理、尚未有補收單號的資料。
- 產生補救明細/補收通知不會修改原繳費單或原 PDF。
- 補救明細/補收通知是補充說明；帳務金額需歸入處理月份學費試算與月結，不可回頭覆蓋原月份單據。
- 補救明細/補收通知不會自動寫入鐘點結算、收據或領據。若同一漏登課程也影響講師鐘點，該筆會進入處理月份的鐘點試算預覽，標示為補發，行政確認後才寫入鐘點結算。
- 補發鐘點認列在處理月份，例如 6 月補登 4 月漏登課程，家長補收單與講師補發鐘點都出現在 6 月流程，不回頭修改 4 月原單據。
- 回看已結算月份時，學費試算會顯示既有結算摘要，不會重新試算授課、預排或補救差異；若要查看 PDF 或寄送狀態，應使用繳費單/已產生單據流程。

## 寄送與 LINE push

- 繳費單：寄給家長，後續支援 LINE push 給家長。
- 收據：寄給家長，後續支援 LINE push 給家長。
- 一般收據：依入會費、常年會費、捐款對象寄送。
- 領據：寄給講師，後續支援 LINE push 給講師。

尚未具備完整資料或權限的項目會顯示錯誤或無資料提示，不會實際寄出。

## 常見注意事項

- 不要在未預覽時直接產生或寄送。
- 發現舊月份錯誤時，不要手改舊單，改用帳務補救。
- 同一批單據寄送前要確認收件人與金額。
- 無法寄送或 LINE push 失敗時，應保留錯誤紀錄供排查。
- 2026-06-11 補充：帳務補救頁面在手機版以單層主要區塊呈現，避免外框層層相套；「補救明細/補收通知」預覽需以分段多行顯示學生、原月份、上課時間、課程、時數單價、原單、狀態、通知單與原因，方便行政人員核對。
- 2026-06-11 更新：領據 PDF 產生已接上 LIFF 預覽確認流程。行政人員需先完成鐘點試算確認寫入，再於領據預覽勾選講師並確認產生 PDF；系統會回寫鐘點結算表 PDF 欄與「待寄送」狀態。領據 Email 與 LINE push 需另外進入寄送預覽、勾選並二次確認後才會寄送。
- 2026-06-11 更新：學費收據 PDF 產生已接上 LIFF 預覽確認流程。行政人員需先確認學費結算表已有收款方式、收據類別與收款日期，再於收據預覽勾選學生並確認產生 PDF；系統會回寫收據 PDF 欄與「待寄送」狀態。收據 Email 寄送與 LINE push 仍需另外預覽確認，尚未開放直接寄送。
## 單據總覽

1. 進入行政作業。
2. 進入「帳務工具」。
3. 選擇處理月份。
4. 點選「單據總覽」。
5. 系統會顯示該月份繳費單、收據、領據、一般收據的總覽。

注意：若只是回看舊月份 PDF 或寄送狀態，應使用「單據總覽」。不要重新執行學費試算或鐘點試算。

2026-06-11 更新：單據總覽改讀 `單據紀錄表`。行政人員查看已產生 PDF、Email 狀態、LINE push 狀態、補發或作廢狀態時，應以此總覽為主，不再分別回頭查學費結算表或鐘點結算表。

## 收據與領據寄送通知

1. 先完成收據或領據 PDF 產生，狀態需為「待寄送」。
2. 進入行政作業，選擇月份。
3. 在收據流程或鐘點流程點選「寄送通知」。
4. 預覽收件人、單號、金額、PDF、Email 狀態與 LINE 狀態。
5. 勾選要通知的對象，並確認卡片內 Email / LINE 子勾選。
6. 確認後系統依勾選通道寄送 Email 或 LINE push，並各自回寫狀態或失敗原因。

注意：沒有 PDF、沒有 Email、或狀態不是「待寄送」的資料會顯示提醒，但在寄送模式下不可勾選。

2026-06-11 更新：收據與領據 Email 寄送後，除了原結算表狀態欄，也會同步更新 `單據紀錄表` 的 Email 狀態、寄送時間與失敗備註。後續補發、作廢與 LINE push 會延續使用同一張中心表。

2026-06-11 更新：收據通知只會讀取 `單據紀錄表` 的「收據」資料；領據通知只會讀取「領據」資料。若只產生了繳費單但尚未產生收據，請不要到收據通知頁找繳費單，應使用學費流程的「寄送通知」或「單據總覽」查看。

2026-06-12 更新：手機行政頁已移除月份欄下方「本月」、「上月」、「已產生單據」快速入口，且各流程分頁也不再放置「已產生單據」按鈕。作廢、補發、重新產生集中在帳務工具的單據總覽卡片上方處理，月份統一使用月份欄選擇。

2026-06-13 更新：行政端入口改為四個 TAB：
- 學費流程：學費試算、產生繳費單、寄送通知。
- 收據流程：收款資料、產生收據、寄送通知、一般收據。
- 鐘點流程：鐘點試算、產生領據、寄送通知。
- 帳務工具：單據總覽、帳務補救、系統工具。

帳務工具內層：
- 單據總覽：查看 PDF、Email、LINE、作廢、補發、重產狀態；先在卡片上方選擇作廢、補發或重產，再勾選符合狀態的單據。
- 帳務補救：點選後才展開補收/退費表單與補救明細。
- 系統工具：稅務專區、查我的 ID。

2026-06-13 更新：學費、收據、鐘點三大流程改為同一種操作方式：
- 點選學費流程、收據流程或鐘點流程分頁時，系統會直接讀取該月份主要資料預覽，不再多放一次 `學費試算`、`收款資料` 或 `鐘點試算` 入口卡。
- 先選擇資料卡片上方的動作按鈕：寫入、產生、寄送通知。
- 畫面會只允許該動作可處理的學生或講師勾選。
- 三大流程進入分頁後即顯示目前完整狀態；寄送通知不再另開查詢頁，按鈕直接依目前卡片的 Email / LINE 狀態啟用或停用。
- 同一筆資料在不同動作下會有不同可處理狀態；例如已套用收款資料的學生，不會再出現在「套用收款資料」可勾選清單，但尚未產生收據 PDF 時仍可在「產生收據」模式勾選。
- 已產生 PDF 的繳費單、收據或領據，不會再出現在產生 PDF 的可勾選清單；後續請改用「寄送通知」或「帳務工具」。
- 寄送通知會在同一卡片內顯示 Email 與 LINE 狀態；兩個通道各自勾選、各自寄送、各自回寫，不會因其中一個缺資料就阻擋另一個。
- 不可處理的資料不會顯示可勾選 checkbox，畫面會以不可處理標示與短原因提醒。
- 卡片主狀態只顯示單號、PDF 或流程狀態；Email / LINE 狀態請看卡片內的 Email / LINE 子通道。

2026-06-13 更新：三大流程改為「最後狀態工作台」。進入學費、收據、鐘點流程時，系統會直接讀取寫入、PDF、Email、LINE 的完整狀態；動作按鈕只針對目前卡片執行，不再先切換到另一個預覽頁查詢狀態。
- 上方流程區不再放多張動作卡片，也不再顯示「可執行」或「預覽確認」等說明。

2026-06-13 更新：行政預覽以學生或講師明細為主，不再顯示開發期狀態提示、資料來源統計卡或重複摘要。可執行項目仍需先預覽、勾選，再按確認。

2026-06-13 更新：繳費單、收據、領據的 Email 與 LINE push 主入口合併為「寄送通知」。系統會依卡片內 Email / LINE 子勾選分別寄送與回寫。

## 收據收款資料套用

1. 進入行政作業，選擇月份。
2. 點選「收據」預覽。
3. 在「收款資料套用」填寫收款方式、收據類別、收款日期。
4. 點選「套用收款資料」。
5. 系統會寫入尚未產生收據 PDF 的學費結算資料。
6. 回到收據預覽，確認資料完整後再勾選產生收據。

注意：已產生收據 PDF 的資料不會被此批次套用覆蓋。

## 試算分頁後續流程

- 學費流程：先開學費試算預覽，在資料卡片上方選擇寫入學費結算、產生繳費單或寄送通知，再勾選可處理對象。
- 收據流程：先開收款/收據預覽，在資料卡片上方選擇套用收款資料、產生收據或寄送通知，再勾選可處理對象。
- 鐘點流程：先開鐘點試算預覽，在資料卡片上方選擇寫入鐘點結算、產生領據或寄送通知，再勾選可處理講師。
- 寄送通知同時顯示 Email 與 LINE push 子通道；行政人員可依狀態勾選一個或兩個通道，系統會分別寄送與回寫。
- 單據查核與版本管理集中到帳務工具的單據總覽、作廢、補發、重新產生流程，不在每個流程分頁重複顯示。

## 單據 Email 預覽資料來源

- 繳費單、收據、領據 Email 預覽以 `單據紀錄表` 為主要來源。
- 若舊資料只有在原結算表留下 PDF，系統會自動補讀：
  - 繳費單：讀取 `學費結算表` 的繳費單 PDF。
  - 收據：只讀取 `單據紀錄表` 中真正的收據 PDF 與待寄送狀態；不可讀取 `學費結算表` 的繳費單 PDF 欄位。
  - 領據：讀取 `鐘點結算表` 的領據 PDF 與待寄送狀態。
- 使用補讀資料寄送成功後，系統會回補 `單據紀錄表`，後續就能在已產生單據與寄送狀態中追蹤。
- 若預覽仍顯示 0 位，請先檢查原結算表是否已有單據編號、PDF 連結、Email 與待寄送狀態。

## 單據總覽顯示

2026-06-13 更新：單據總覽放在「帳務工具」內，不在每個流程頁重複顯示。總覽會依單據類型顯示 PDF 已產生數、Email 待寄送/已寄送/失敗、LINE 未推播/已推播與金額摘要；明細會列出對象、單號、PDF、Email、LINE、寄送時間、連結與備註。
## 單據作廢、補發與重新產生

「單據總覽」是帳務工具的主要工作面板；作廢、補發、重新產生按鈕位於單據卡片上方，需先選擇操作，再勾選符合狀態的單據後才能執行。

- 作廢：用於單據內容或金額已不應使用時。作廢不刪除舊 PDF，也不刪除 `單據紀錄表`，只留下作廢狀態與原因，方便日後追查。
- 補發：用於家長或講師沒有收到、需要再寄一次同一份 PDF 時。補發不改金額、不改內容、不改單據編號。
- 重新產生：用於尚未寄出的單據需要修正 PDF 時。若單據已寄出，應先作廢原單，再建立新版或更正版。

行政操作原則：凡是會改狀態、產生 PDF、寄送 Email 或 LINE push 的功能，都必須先看到預覽，確認後才執行。
## LINE push 前置檢查

LINE push 已開放正式執行；行政人員仍可先進入「單據總覽」查看準備度：

- 繳費單：推播對象是家長/學生。
- 收據：推播對象是家長/學生。
- 領據：推播對象是講師本人。
- 若顯示缺 LINE ID，代表該對象尚未有可推播的 LINE User ID，不能推播。
- 若顯示缺 PDF，需先完成產生單據。
- 若單據已作廢，不可再推播。

正式開放後，LINE push 仍需先看預覽、勾選對象、二次確認後才執行。
## LINE push 寄送

LINE push 第一版採純文字訊息與 PDF 連結，不使用圖文卡片。

操作流程：
1. 進入對應流程。
2. 點選 `寄送繳費單 LINE`、`寄送收據 LINE` 或 `寄送領據 LINE`。
3. 檢查預覽清單中的對象、金額、單據編號、PDF 與 LINE ID 狀態。
4. 勾選本次要推播的項目。
5. 按確認按鈕，完成二次確認後才會推播。

推播對象：
- 繳費單：家長 LINE。
- 收據：家長 LINE。
- 領據：講師本人 LINE。

若顯示缺 LINE ID，請先補齊學生基本資料表 D 欄或講師名單 B 欄，再重新預覽。

既有領據若已在鐘點結算表產生 PDF，但尚未同步到 `單據紀錄表`，系統會在領據 LINE push 預覽中補讀；確認推播時會先同步 `單據紀錄表` 再推播。

工具區中的 `作廢單據` 已開放正式流程。行政人員需先預覽、勾選單據、二次確認，並輸入作廢原因。系統只會將單據標記為作廢，不會刪除 PDF 或資料列。

工具區中的 `補發單據` 已開放正式流程。行政人員需先預覽、勾選單據、二次確認，並輸入補發方式：`Email`、`LINE` 或 `both`。補發只會重寄同一份 PDF，不會改金額、內容或單據編號。

工具區中的 `重新產生單據` 已開放正式流程。行政人員需先預覽、勾選單據、二次確認，並輸入重新產生原因。只有未 Email 寄送、未 LINE 推播、未作廢且已有 PDF 的繳費單、收據、領據可以重新產生；系統會移除舊 PDF、重新產生新 PDF，並在 `單據紀錄表` 備註保留重產原因與舊 PDF 連結。已對外寄送或推播的單據不可直接覆蓋，需先作廢，再建立新版。

## 單據 PDF 檔名與雲端資料夾

- 新產生的繳費單、收據、領據與一般收據 PDF 會以單據編號開頭，方便在 Google Drive 依名稱排序。
- 範例：`R_2026_06_001_繳費單_學生姓名.pdf`、`A_2026_06_001_領據_講師姓名.pdf`。
- 現階段維持既有 Drive 分類資料夾，不另外每月自動建立子資料夾。
- 舊單據仍可能維持舊檔名；系統查詢會相容舊檔名，不需要手動搬移或重新命名。
- 行政預覽卡片若顯示 `PDF：已產生`，代表系統已有 PDF 連結；完整 Drive 連結請以試算表或正式單據紀錄為準，畫面不再顯示長網址。

## 帳務補救補登防重與預覽

- 補登帳務補救時，同一學生、同一課程、同一原上課日期、同一開始/結束時間、同一補救類型不可重複建立。
- 若系統提示已有相同帳務補救紀錄，請先檢查 `學費調整紀錄表` 對應列，不要再次送出。
- 補登成功後，畫面會在表單下方顯示最近補登結果。
- 表單下方的 `補救明細/補收通知` 可用來查看本月待處理補救資料，確認是否已成功進入試算流程。
- 繳費單、收據、領據等明細若已有 PDF，畫面會顯示 `查看 PDF` 文字連結，可直接開啟檢查，不必進 Google Drive 搜尋。

## 一般收據

一般收據用於入會費、常年會費與捐款，不屬於學生課程費收據。

操作流程：
1. 進入 `行政作業` → `收據流程`。
2. 點選 `一般收據`。
3. 填寫收據日期、姓名/單位、類別、收款方式與金額。
4. 點選 `產生一般收據 PDF`。
5. 二次確認後，系統會產生 PDF，寫入 `一般收據紀錄` 與 `會計日記帳`。
6. 產生完成後，可在預覽明細中點選 `查看 PDF`。

寄送 Email：
1. 進入 `一般收據`。
2. 點選 `寄送一般收據 Email`。
3. 檢查 PDF、Email、類別與金額。
4. 勾選要寄送的項目。
5. 二次確認後寄送。

注意：目前一般收據 LINE push 尚未開放，需先定義會員/捐款者 LINE ID 欄位來源。
