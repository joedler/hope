# 行政人員操作手冊

最後更新：2026-06-05

## 操作原則

行政端所有高風險動作都必須先看預覽，再確認執行。

高風險動作包含：

- 寫入試算結果
- 產生繳費單
- 產生收據
- 產生領據
- 寄送 Email
- LINE push
- 建立帳務補救紀錄
- 作廢、補發、重新產生單據

## 行政端 Tab

### 試算

用途：計算指定月份的學費與講師鐘點。

流程：

1. 選擇月份。
2. 點選學費試算或鐘點試算。
3. 查看預覽結果。
4. 確認無誤後才寫入。

現況：

- 目前 LIFF 先提供前端預覽卡。
- 前端會優先讀取只讀型 `adminPreview` API。
- `adminPreview` 不會寫入、產生單據、寄送 Email 或 LINE push。
- 預覽卡會顯示資料來源統計，例如授課紀錄、預排紀錄、結算表、帳務補救筆數。
- 統計筆數只用於檢查資料來源是否存在，不等於正式結算金額。
- 每個資料來源會顯示狀態：有資料、本月無資料、找不到分頁、讀取失敗。
- 若出現「找不到分頁」或「讀取失敗」，不要繼續進行結算或產生單據，需先排查 Google Sheets 分頁名稱、權限或欄位設定。
- 學費試算預覽會先彙總待試算授課、預收下月預排、預收核對差異與帳務補救金額。
- 若預覽提示該月份已有學費結算資料，代表這可能是舊月份回看情境；回看舊繳費單應使用「已產生單據」，不要重新試算。
- 鐘點試算預覽會先彙總待試算授課、講師時數、應付總額、扣繳稅額、補充保費與實發金額。
- 若預覽提示該月份已有鐘點結算資料，代表這可能是舊月份回看情境；回看舊領據或結算應使用「已產生單據 / 領據紀錄」，不要重新試算。
- 確認寫入按鈕以停用狀態顯示，待正式預覽 API 與二次確認完成後開放。
- 切換行政 Tab 時會清除上一個預覽，避免誤讀不同功能的內容。

### 繳費單

用途：產生並寄送學生繳費通知。

流程：

1. 選擇月份與學生範圍。
2. 產生繳費單預覽。
3. 檢查學生、課程、預收/後收、補收/退費、總金額。
4. 確認後產生繳費單。
5. 可寄送 Email 給家長。
6. 後續新增 LINE push 給家長個人 LINE。
7. 可查看已產生單據。
8. 後續增加重新產生、作廢、補發。

現況：

- 繳費單預覽會讀取學費結算表，顯示學生、單據編號、總金額、課程數與 PDF 產生狀態。
- 繳費單預覽不會重新試算學費，也不會產生 PDF、寫入 PDF URL、寄送 Email 或 LINE push。
- 若看不到繳費單資料，需先確認該月份是否已完成學費試算確認寫入。

### 收據

用途：針對已繳費資料產生與寄送收據。

流程：

1. 選擇月份或繳費單。
2. 產生收據預覽。
3. 確認付款資訊、學生、金額、收據內容。
4. 確認後產生收據。
5. 可寄送 Email 給家長。
6. 後續新增 LINE push 給家長個人 LINE。

一般收據：

- 適用入會費、常年會費、捐款。
- 需要預覽與寄送功能。
- 後續可再依稅務需求擴充。

### 領據

用途：針對講師鐘點費產生與寄送領據。

流程：

1. 選擇月份與講師。
2. 產生領據預覽。
3. 檢查授課紀錄、預排核銷、補充鐘點、總金額。
4. 確認後產生領據。
5. 可寄送 Email 給講師。
6. 後續新增 LINE push 給講師個人 LINE。

### 帳務補救

用途：處理漏登、短收、超收、補收、退費。

原則：

- 不直接修改舊繳費單。
- 必須建立補救紀錄。
- 必須關聯原繳費單。
- 必須在後續月份反映補收或退費。
- 補收/退費要併入處理月份月結，不重新產生或覆蓋原繳費單；必要時可另產生補救明細/補收通知作為說明。

流程：

1. 選擇補收或退費。
2. 選擇原錯誤月份。
3. 選擇處理月份。
4. 選擇學生。
5. 選擇課程。
6. 填入原上課日期與時間。
7. 系統帶出時數、單價、預覽金額。
8. 選擇關聯原單號。
9. 填寫原因。
10. 預覽補救影響。
11. 確認後建立補救紀錄。

補救明細/補收通知流程：

1. 在帳務補救分頁選擇處理月份。
2. 點選「預覽補救明細」。
3. 檢查待處理補收的學生、金額、筆數與關聯原單號。
4. 確認後點選「確認產生」。
5. 系統產生補救明細/補收通知 PDF，單號格式為 `ADJ_YYYY_MM_###`。
6. 系統回寫學費調整紀錄表的補收單號、補收 PDF、補收單狀態，並將狀態改為已產生補收單。

注意：

- 補救明細/補收通知只處理調整類型為補收、狀態為待處理、尚未有補收單號的資料。
- 產生補救明細/補收通知不會修改原繳費單或原 PDF。
- 補救明細/補收通知是補充說明；帳務金額需歸入處理月份學費試算與月結，不可回頭覆蓋原月份單據。
- 補救明細/補收通知不會自動寫入鐘點結算、收據或領據。若同一漏登課程也影響講師鐘點，該筆會進入處理月份的鐘點試算預覽，標示為補發，行政確認後才寫入鐘點結算。
- 補發鐘點認列在處理月份，例如 6 月補登 4 月漏登課程，家長補收單與講師補發鐘點都出現在 6 月流程，不回頭修改 4 月原單據。
- 回看已結算月份時，學費試算會顯示既有結算摘要，不會重新試算授課、預排或補救差異；若要查看 PDF 或寄送狀態，應使用繳費單/已產生單據流程。

## 寄送與 LINE push

- 繳費單：寄給家長，後續支援 LINE push 給家長。
- 收據：寄給家長，後續支援 LINE push 給家長。
- 一般收據：依入會費、常年會費、捐款對象寄送。
- 領據：寄給講師，後續支援 LINE push 給講師。

目前尚未接正式寄送 API 的項目，只會顯示「尚未開放」提示，不會實際寄出。

## 常見注意事項

- 不要在未預覽時直接產生或寄送。
- 發現舊月份錯誤時，不要手改舊單，改用帳務補救。
- 同一批單據寄送前要確認收件人與金額。
- 無法寄送或 LINE push 失敗時，應保留錯誤紀錄供排查。
