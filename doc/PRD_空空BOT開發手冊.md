# PRD_空空BOT開發手冊

最後更新：2026-06-05

## 產品定位

空空BOT 是「撐出空間教育關懷協會」用於講師授課登記、課程預排、預排核銷、行政單據與帳務補救的 LINE + LIFF + GAS + Google Sheets 工作台。

核心目標不是做大量功能入口，而是降低每月行政與講師操作錯誤，讓高風險作業都有預覽、確認、防呆與追溯紀錄。

## 使用者角色

- 講師：登記已完成授課、預排課程、核銷預排課程、查看自己的狀態。
- 行政人員：試算學費與鐘點、產生與寄送單據、處理收據/領據、帳務補救、權限與綁定排查。
- 家長：接收繳費單、收據與相關通知。
- 系統維護者：維護 GAS、LIFF、Google Sheets、LINE Messaging API 與資料規則。

## 核心設計原則

- 使用者入口以 LIFF 工作台為主，LINE 文字指令保留作為輔助與除錯。
- 所有會寫入、產生單據、寄送 Email、LINE push 的功能，一律採用「預覽 → 確認 → 執行」。
- 不直接覆蓋舊單據。修正舊月份錯誤時，使用帳務補救紀錄追溯處理。
- 資料應以 Google Sheets 為現階段主資料庫，但程式不可依賴脆弱欄位位置，應盡量用表頭與欄位名稱。
- 權限必須由後端再次檢查，前端隱藏入口不能視為安全控管。

## 講師端功能

- 我的狀態：顯示身份別、權限、本月已登記授課、待核銷預排、可登記學生、可登記課程。
- 授課登記：已上完課後，登錄學生、課程、日期、開始時間、結束時間。
- 預排課程：預先登記下個月或未來課程。
- 預排核銷：課程時間結束後，確認有上課或未上課。
- 多筆新增：以同一學生為主體，最多 5 筆，支援同學生同課程不同時間或同學生不同課程。

## 行政端功能架構

行政端應採用 Tab 工作台，不再以一堆大格子鋪滿畫面。

建議 Tab：

- 試算
- 繳費單
- 收據
- 領據
- 帳務補救
- 稅務專區
- 權限/查 ID

## 行政端高風險規則

- 試算：學費試算、鐘點試算都先預覽，確認後才寫入。
- 繳費單：產生前先預覽，確認後才產生。後續支援寄送 Email、LINE push 給家長、查看已產生單據、重新產生、作廢、補發。
- 收據：產生前先預覽，確認後才產生。保留 Email 寄送，新增 LINE push 給家長。一般收據處理入會費、常年會費、捐款，至少需有預覽與寄送。
- 領據：產生前先預覽，確認後才產生。保留寄送功能，新增 LINE push 給講師。
- 帳務補救：處理漏登、短收、超收、補收、退費，必須關聯原繳費單，不直接改舊單。

## 行政預覽 API

- `adminPreview` 是只讀 API，只能回傳預覽資料，不可寫入、產生 PDF、寄送 Email 或 LINE push。
- `adminTask` 屬於執行型 API，不可由高風險入口直接呼叫。
- 前端行政預覽優先呼叫 `adminPreview`，若 API 尚未部署或失敗，退回本機預覽卡，避免操作中斷。
- 預覽資料需包含：標題、月份、摘要、檢查項目、資料來源統計、目前狀態、下一步按鈕文字。
- 資料來源統計只用於檢核資料是否存在，例如授課紀錄、預排紀錄、學費結算、鐘點結算、帳務補救筆數；不代表正式試算結果。
- 資料來源統計需回傳狀態，不可只回傳數字。狀態包含：`ok` 有資料、`empty` 本月無資料、`missing` 找不到分頁、`error` 讀取失敗。
- `0` 筆資料必須區分為「本月無資料」或「找不到分頁 / 讀取失敗」，避免行政人員誤以為資料正常。

## 時間欄位規格

- 顯示區一律為 24 小時制 `HH:MM`。
- 不允許顯示 `上午`、`下午`、`AM`、`PM` 或只顯示 `1:00`。
- 分鐘間隔固定為 `00 / 15 / 30 / 45`。
- 手機端可用原生時間選擇器；電腦端使用可控小面板。
- 詳細決策見 `doc/時間欄位設計決策紀錄.md`。

## 後續開發順序建議

1. 行政端改為 Tab 工作台。
2. 將試算、繳費單、收據、領據、帳務補救都整理為預覽式流程。
3. 建立單據狀態與寄送紀錄。
4. 建立 LINE push 收件人綁定與錯誤提示。
5. 擴充測試清單，避免月底結算重複出錯。
