# GoogleSheets資料表欄位對照

最後更新：2026-06-05

## 文件用途

本文件用來記錄 Google Sheets 中各工作表的用途、欄位、寫入來源與注意事項。實際欄位需在後續盤點正式試算表後補齊。

## 通用原則

- 程式應盡量依表頭名稱讀寫，不依賴固定欄位位置。
- 系統產生欄位不可由使用者任意手改。
- 高風險寫入前需先預覽。
- 補救舊資料時，優先新增補救紀錄，不直接覆蓋舊單據。

## 已知工作表

### 講師資料 / 綁定資料

用途：

- LINE User ID 綁定講師或行政身份。
- 判斷是否可進入行政作業。

寫入來源：

- LINE 文字指令綁定。
- 行政人員設定。

注意事項：

- LINE User ID 必須正確。
- 前端顯示權限之外，後端 API 也要檢查。

### 學生與課程資料

用途：

- 定義學生可上的課程。
- 定義課程收費模式：預收或後收。
- 定義單價。

寫入來源：

- 行政維護。

注意事項：

- LIFF 講師端選學生後，只能顯示該學生課程。
- 課程收費模式目前只有預收、後收。

### 授課紀錄

用途：

- 記錄講師已完成授課。

寫入來源：

- 講師授課登記。
- 預排核銷有上課後轉入或更新。
- 帳務補救可能用於追溯漏登，但不應直接破壞原紀錄。

注意事項：

- 不可登記未來時間為已授課。
- 需檢查重疊與重複。

### 預排紀錄

用途：

- 記錄未來預排課程。
- 提供預排核銷清單。

寫入來源：

- 講師預排課程。

注意事項：

- 預排可顯示未來課程，但未到課程結束時間不可核銷。
- 同筆核銷後不可重複核銷。

### 請假紀錄

用途：

- 記錄請假資料與額度變化。

現況：

- 目前先不放入 LIFF，保留行政授權人員用 LINE 對話框或既有流程處理。

### 學費試算 / 月結資料

用途：

- 計算學生當月應繳金額。
- 處理預收、後收、補收、退費。

寫入來源：

- 行政端學費試算確認後寫入。

注意事項：

- 不可試算即直接寫入。
- 預覽需確認後才寫入。
- 需避免重複試算造成重複資料。

### 鐘點試算

用途：

- 計算講師鐘點與補充項目。

寫入來源：

- 行政端鐘點試算確認後寫入。

注意事項：

- 需與授課紀錄、預排核銷一致。

### 繳費單紀錄

用途：

- 記錄已產生繳費單。
- 支援查看、寄送、補發、作廢、重新產生。

寫入來源：

- 行政端繳費單確認產生。

注意事項：

- 學生姓名、單據編號、金額欄位不可錯位。
- 需保留產生時間與狀態。

### 收據紀錄

用途：

- 記錄學費收據與一般收據。

寫入來源：

- 行政端收據功能。

注意事項：

- 一般收據包含入會費、常年會費、捐款。
- 寄送與 LINE push 需保留紀錄。

### 領據紀錄

用途：

- 記錄講師鐘點領據。

寫入來源：

- 行政端領據功能。

注意事項：

- LINE push 對象為講師。

### 學費調整紀錄 / 帳務補救

用途：

- 記錄補收、退費、漏登補救。
- 關聯原繳費單。
- 在處理月份反映補救金額。

寫入來源：

- 行政端帳務補救。

注意事項：

- 不直接修改舊繳費單。
- 必須保留原因、原錯誤月份、處理月份、關聯單號。
- 補救明細/補收通知產生後需回寫補收單號、補收 PDF、補收單狀態。
- 補收單號格式為 `ADJ_YYYY_MM_###`。
- 補收單號與補收 PDF 屬於補救明細/補收通知，不可覆蓋原繳費單號或原 PDF；實際月結金額仍歸入處理月份學費結算。
- 補救紀錄是來源事件；學費結算、收據、鐘點費、領據、稅務資料需依各自流程預覽確認後才寫入。
